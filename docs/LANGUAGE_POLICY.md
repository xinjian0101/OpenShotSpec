# Repository Language Policy

OpenShotSpec maintains its public documentation, examples, schemas, issue templates, and maintenance records in English.

## Scope

The policy applies to:

- README files
- documentation pages
- JSON Schema titles and descriptions
- example records
- test fixture labels
- changelog and maintenance entries
- contribution guidance

## Compatibility exception

Stable field names, extension namespaces, external identifiers, and imported source values are not renamed solely for language consistency when doing so would break compatibility or misrepresent source data.

## Contribution requirements

- Write new public documentation in clear English.
- Explain specialized terminology on first use.
- Keep examples synthetic and portable.
- Avoid unexplained abbreviations.
- Preserve the meaning of shared fields across translations and adapters.
- Record any language-related compatibility exception in the relevant change note.

## Review checklist

A documentation review should confirm that headings, prose, example values, warnings, and maintenance notes are English and that links point to current files.

## Maintenance cycle 4

This policy records iteration 74 of the cross-repository English conversion cycle. Existing OpenShotSpec content was already English, so the iteration establishes a durable rule for future contributions.
